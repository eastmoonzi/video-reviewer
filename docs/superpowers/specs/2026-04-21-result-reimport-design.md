# 评估结果重新导入 — 设计文档

**日期：** 2026-04-21  
**状态：** 已确认

---

## 背景

当前导出的 Excel 只包含人工评分数据，无法作为完整备份。用户需要：
1. 在新设备/新浏览器中恢复评审进度
2. 将多个评审员各自导出的结果合并到同一工作区

---

## 目标

- 导出文件附带任务重建所需的完整信息
- 导入时自动识别"评分结果文件"，无需额外入口
- 支持恢复已有任务评分，也支持新建缺失任务

---

## 设计

### 一、导出增强

三种模式（segment / profile / audiovisual）的导出函数各自追加 4 个新列：

| 列名 | 内容 | 说明 |
|------|------|------|
| `__任务原始ID` | `task.rawId` | 精确匹配任务标识 |
| `__视频标题` | `task.id`（显示名） | 重建时还原任务显示名 |
| `__视频时长` | `task.video_duration`（秒，整数） | 重建时还原视频时长 |
| `__模型输出JSON` | `JSON.stringify(model_output)` | 重建时还原模型输出内容 |

`__` 前缀区分机器列与人工可读列，同时作为文件类型识别标记。

### 二、导入识别与路由

在现有 `confirmImport()` 流程中，Excel 解析完成后检测列名：
- 存在 `__模型输出JSON` 列 → 判定为"评分结果文件" → 调用 `importResultsFromExcel(rows)`
- 否则 → 走原有逻辑（原始数据导入）

`importResultsFromExcel` 内部通过列名检测审核模式：

| 判断依据 | 模式 |
|----------|------|
| 含"时间段切分评分" | `segment` |
| 含"叙事类型评分" | `profile` |
| 含"总体质量评分"列 | `audiovisual` |

匹配键：`视频URL` + `模型名称`（与多文件合并逻辑一致）。

### 三、冲突处理

**任务已存在且已有评分时**，弹出确认弹窗：
- 显示冲突条目数量
- 选项：**覆盖所有** / **跳过所有**
- 批量处理，不逐条询问

**任务不存在时**，用导出列重建任务：
1. `__模型输出JSON` → `JSON.parse` → `normalizeModelOutput()` 还原模型输出
2. `__任务原始ID` / `__视频标题` / `__视频时长` / `视频URL` 填充元数据
3. 评分/备注直接写入对应 review 对象
4. 追加到当前工作区任务列表末尾

### 四、导入结果反馈

导入完成后显示 toast：

```
已恢复 N 条评分，新建 M 个任务，跳过 K 条
```

---

## 不在范围内

- 不支持跨工作区合并（仅操作当前工作区）
- 不支持逐条冲突确认（批量覆盖或批量跳过）
- 不支持导入旧格式（不含 `__模型输出JSON` 列）的结果文件作为任务新建来源
